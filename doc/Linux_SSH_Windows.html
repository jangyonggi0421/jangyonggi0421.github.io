SSH 인증키 접속 설정

시스템 환경
  • OS: Rocky Linux 9.7 (Blue Onyx)
  • Kernel: Linux 5.14.0-611.36.1.el9_7.x86_64
  • 접속 OS: Windows 11 Pro (25H2, 26200.7922)

1. SSH 설치 확인

openssh-server 패키지가 설치되어 있는지 확인합니다.

# 1. 패키지 업데이트
sudo dnf update -y

# 2. 설치 확인
rpm -qa | grep openssh-server
# 결과 예시: openssh-server-8.7p1... 가 나오면 설치됨

# 2-1. 미설치 시 설치
sudo dnf install openssh-server -y

2. SSH 서비스 활성화

# 1. 서비스 활성화 및 즉시 시작
sudo systemctl enable --now sshd

# 2. 상태 확인 (Active: active(running) 확인)
sudo systemctl status sshd

3. 방화벽 설정

# 1. 서비스 항목에 ssh 포함 여부 확인
sudo firewall-cmd --list-all

# 1-1. 항목이 없을 경우 추가
sudo firewall-cmd --permanent --add-service=ssh

# 2. 규칙 즉시 반영
sudo firewall-cmd --reload

4. SSH 인증키 접속 준비

Rocky Linux 9부터는 보안 강화를 위해 root 계정의 비밀번호 접속이 기본적으로 금지됩니다.

Linux 서버 설정 변경

# 1. 디렉토리 생성 및 권한 설정
mkdir -p ~/.ssh
chmod 700 ~/.ssh
restorecon -Rv ~/.ssh

# 2. sshd_config 설정 수정
su -
vi /etc/ssh/sshd_config

# 수정 사항 (주석 해제 및 설정 확인):
# 45라인: PubkeyAuthentication yes
# 49라인: AuthorizedKeyFile .ssh/authorized_keys
# 65라인: PasswordAuthentication yes (키 등록 전까지 한시적 허용)

5. Windows 클라이언트 설정 및 접속

Windows PowerShell에서 작업을 수행합니다.

# 1. SSH 키 생성
ssh-keygen -t rsa -b 4096

# 2. 생성된 공개키(.pub) 내용 복사
Get-Content C:\Users\<사용자명>\.ssh\<키이름>.pub | Set-Clipboard

# 3. 서버에 공개키 등록
ssh <User>@<IP주소>
echo "복사한 키 내용" >> ~/.ssh/authorized_keys
exit

# 4. 보안 강화 (비밀번호 접속 차단)
# /etc/ssh/sshd_config 파일에서 PasswordAuthentication no 로 수정

# 5. 서비스 재시작
sudo systemctl restart sshd
