Git - GitHub 계정 연동, 한글 깨짐 해결

1. Git - GitHub 계정 연동

"Git Bash" 실행
  1. GitHub 계정 설정

    Git Bash에서 다음과 같이 입력하여 전역 사용자 정보를 설정합니다.

    # 본인의 이름(닉네임) 설정
git config --global user.name "본인의 닉네임"

# GitHub 가입 시 사용한 이메일 설정
git config --global user.email "GitHub 가입 이메일"
  2. 리포지토리 연동

    기존 리포지토리가 있다면 클론(Clone)하여 연동하고, 없다면 생성 후 진행합니다.

    git clone https://github.com/“repository 이름”
  3. 브라우저 인증

    명령어 입력 후 나타나는 Sign in with your browser 버튼을 클릭하여 로그인하면 연동이 완료됩니다.

2. 터미널에서 한글 깨짐 방지

Git Bash에서 한글이 올바르게 표시되도록 다음 명령어들을 입력합니다.

  1. 파일명 깨짐 예방 
    git config --global core.quotepath false
  2. 출력 인코딩 UTF-8 설정 
    export LC_ALL=ko_KR.UTF-8 
export LANG=ko_KR.UTF-8
  3. Git Bash 화면 설정 (가독성 업그레이드)
    • 폰트 변경: Git Bash 창에서 마우스 오른쪽 버튼 클릭 > Options > Text > Font 버튼 클릭 후 'Cascadia Code'와 같은 개발용 폰트로 변경하면 한글/영문 가독성이 좋아집니다.
    • 투명도 조절: Window 메뉴에서 Transparency를 조절하면 작업 환경을 커스터마이징할 수 있습니다.
