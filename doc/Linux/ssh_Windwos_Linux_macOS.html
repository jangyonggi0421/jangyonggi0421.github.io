SSH - Windows, Linux 에서 macOS 접속

접속 환경 (OS):
  • Windows 11 Pro
  • Rocky Linux 9.7 (5.14.0-611.5.1.el9_7.x86_64) >> macOS Tahoe 26.4
macOS

macOS에는 Linux의 systemctl에 해당하는 명령어가 없으며, launchctl이 그 역할을 대체합니다.

  • 서비스 상태 확인: sudo launchctl list (목록에서 '-'는 실행 중이 아님을 의미)
  • 서비스 시작: sudo launchctl start
  • 서비스 중지: sudo launchctl stop

공통 설정

1단계. macOS SSH 설정

# 1. sshd_config 파일 열기
# vi /etc/ssh/sshd_config

# 2. 설정 변경 (i 키로 입력 모드 전환)
# - #PubkeyAuthentication yes 에서 주석 제거 (공개키 인증 활성화)
# - PasswordAuthentication no (비밀번호 로그인 비활성화)
# - #AuthorizedKeysFile .ssh/authorized_keys 주석 제거

# ESC 입력 후 :wq 로 저장

2단계. SSH 인증키 생성 (Windows, Linux 공통)

# 1. SSH 인증키 생성 (RSA 4096비트)
ssh-keygen -t rsa -b 4096

# -t: 키 타입(RSA)
# -b: 비트 길이(4096)
# 프롬프트 안내:
# - 파일 경로 지정: Enter (기본값)
# - Passphrase(암호): Enter (공백 가능)

# 2. 생성된 키 파일 확인
ls -al
# id_rsa (비공개 키 - 노출 금지)
# id_rsa.pub (공개 키)

플랫폼별 키 등록

Windows (PowerShell)

SSH 인증키 등록

# 1. 생성된 공개키(.pub) 내용을 클립보드에 복사
PS > Get-Content $env:USERPROFILE\.ssh\<키 이름>.pub | Set-Clipboard

# 2. 기존 비밀번호 방식으로 접속하여 공개키 등록
PS > ssh <User>@<ip주소>

# 3. 서버 접속 후 클립보드 내용 붙여넣기
echo "복사한 키 내용" >> ~/.ssh/authorized_keys
exit
Linux (Terminal)

SSH 인증키 등록 및 접속

# 1. 공개키 등록 (ssh-copy-id 도구 사용)
ssh-copy-id <User>@<ip주소>
# 패스워드 입력 후 등록 완료

# 2. SSH 접속 테스트
ssh <User>@<ip주소>
# 키 생성 시 암호를 설정하지 않았다면 즉시 로그인됨

최종 반영 및 접속

macOS 서비스 재시작

# 설정을 반영하기 위해 SSH 서비스 재시작
sudo launchctl stop sshd
sudo launchctl start sshd

최종 접속 명령어

ssh [원격서버계정]@[원격지IP]
# 예) ssh user@123.123.11.1
