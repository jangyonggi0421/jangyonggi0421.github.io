SSH 키 등록 오류 발생 시

오류 내용

@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@ @ WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED! @ @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@ IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY! ... 중략 ... Host key for <IP주소> has changed and you have requested strict checking. Host key verification failed.
SSH가 이전에 저장해 둔 서버의 호스트 키와, 지금 접속하려는 서버의 호스트 키가 다를 때 발생합니다.

조치법

  1. 알려진 호스트 키 삭제

    사용자의 로컬 컴퓨터에 저장된 known_hosts 파일에서 특정 IP 주소와 관련된 이전 호스트 키를 삭제합니다.

    ssh-keygen -R 192.168.0.1
  2. SSH 키 다시 등록

    ssh-copy-id 명령어를 사용하여 공개키 내용을 서버에 자동으로 추가합니다.

    ssh-copy-id [원격서버계정]@[원격지IP]

# 예시:
# ssh-copy-id user@123.123.11.1

    ※ 패스워드를 입력하면 공개키가 서버의 ~/.ssh/authorized_keys 파일에 등록됩니다.

  3. sshd 서비스 재시작

    설정을 적용하기 위해 각 운영체제에 맞는 명령어로 서비스를 재시작합니다.

    [Linux] 
    sudo systemctl stop sshd
sudo systemctl start sshd
    [macOS] 
    sudo launchctl stop sshd
sudo launchctl start sshd
  4. 최종 접속 테스트 
    ssh [원격서버계정]@[원격지IP]
