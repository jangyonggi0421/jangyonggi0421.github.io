Linux 에 NVidia 그래픽 카드 드라이버 설치

CUDA 등을 이용하기 위한 Nvidia 그래픽 카드 드라이버 설치
현재 커널 버전 : 5.14.0-611.5.1.el9_7.x86_64

1. 기본 설치 과정

  1. 커널 업데이트 및 개발 도구 설치 
    # root 로그인
su

# 커널 업데이트
sudo dnf update -y

# 개발 도구 설치
sudo dnf groupinstall "Development Tools" -y
sudo dnf install kernel-devel kernel-headers -y
  2. ELRepo 저장소 등록 및 드라이버 설치 
    # ELRepo 저장소 등록
sudo rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org

# 드라이버 설치
sudo dnf install https://www.elrepo.org/elrepo-release-9.el9.elrepo.noarch.rpm -y
sudo dnf install kmod-nvidia -y
  3. 재부팅 및 확인
    • 시스템을 재부팅한 후 nvidia-smi 명령어를 실행합니다.
    • 결과: GPU 정보, 드라이버 버전, CUDA 버전이 화면에 출력되면 설치 성공입니다.

2. 오류 해결 (Troubleshooting)

현재 커널 확인 명령어: uname -r
(확인된 버전: 5.14.0-611.5.1.el9_7.x86_64)

1) 부팅 시 검은 화면(Black Screen) 오류

  • 현상: 부팅 중 Crash recovery kernel arming 메시지와 함께 멈춤.
  • 원인: Secure Boot 활성화 및 특정 최신 커널과 하드웨어(GPU/kdump) 간의 메모리 주소 충돌.
  • 해결책:
    1. BIOS 설정: F2 진입 -> Security -> Supervisor Password 설정 -> Secure Boot Control을 OFF로 변경.
    2. 커널 선택: 최신 버전 대신 하드웨어 호환성이 검증된 5.14.0...5.1 버전으로 부팅 시도.
    3. kdump 비활성화: 불필요한 메모리 점유 및 충돌 방지를 위해 서비스 중지.
    sudo systemctl disable --now kdump

2) NVIDIA 드라이버 저장소 충돌 해결

  • 현상: ELRepo(580 버전)와 RPM Fusion(570 버전) 패키지 간의 파일 소유권 충돌 발생.
  • 원인: 서로 다른 저장소의 드라이버 라이브러리 파일들이 같은 경로(/usr/lib64/)에서 충돌.
  • 해결책: 기존 잔재를 완전히 제거하고 단일 저장소(RPM Fusion)로 통합합니다. 
    # 1. 기존 충돌 패키지 강제 삭제
sudo dnf remove *nvidia* *elrepo* -y

# 2. 필수 저장소(EPEL, RPM Fusion) 등록
sudo dnf install epel-release https://mirrors.rpmfusion.org/nonfree/el/rpmfusion-nonfree-release-9.noarch.rpm -y

3) 커널 맞춤형 드라이버 빌드

특정 커널 버전에서 드라이버가 인식되지 않을 경우, 현재 커널 버전에 맞춰 드라이버를 다시 빌드해야 할 수 있습니다. kernel-devel과 kernel-headers가 현재 실행 중인 커널 버전과 일치하는지 반드시 확인하십시오.
